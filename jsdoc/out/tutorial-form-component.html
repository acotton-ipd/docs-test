Pages

add new page: add_page_icon.svg calls choosePage() which opens Page Chooser (dialog containing file.html template)

Pages can also be added by drag-and-dropping a pdf over the #file-drop-zone (currently the drop zone is set to the entire form.html template).

Drag-and-drop opens the component (PDF viewer) directly.

The is an overlay which is made visible, not an mdDialog.

Adding Pages:

  1. add button
    1. calls choosePage
  2. opens Page Chooser / file.html in mdDialog
    1. dialog promise .then openPanel(file)
      1. if (file), open into PDF Viewer when Page Chooser dialog is closed (???)
  3. choose existing page or add new page
    1. "Browse File" button opens file browser
    2. uses angular-file-model plugin
    3. chosen file > file-model > $ctrl.file
    4. WATCHes for change in file
    5. calls $mdDialog.hide(this.file) //passes filename to hide
    6. after hide, then openPanel(file,null)
    7. openPanel calls PdfService which loads PDF into PDF Viewer

Selected page in is opened into PDF Viewer by: onOpen > openDocument > openPanel > PdfService passes document data from Firebase to openDocument openPanel takes file object from DOM
